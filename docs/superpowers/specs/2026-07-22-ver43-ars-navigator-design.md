# Ver 4.3 (dform2) 재구축 — 음성ARS 메뉴 네비게이터

작성 2026-07-22 · 대상 `목업/index.html`(dform2 덱) + `목업/app.js`(배선)

## 개념
Ver 4.3(스킴 `dform2`)를 **서류제출 폼 → 음성ARS 메뉴 네비게이터("보이는 ARS")**로 전면 개편.
`음성ARS 메뉴구조도.jpg`를 그대로 옮긴 기존 `window.IVR` 대→중→소 트리를 데이터로 재사용하고,
`#dform2-phone` 스코프의 독립 드릴다운 렌더러 + 팝업 + 완료화면을 새로 구축한다.
(형제 시안 dform=Digital Form, dform3=Digital Form_v0.1 은 변경하지 않는다.)

## 비주얼 = "라이트 넘버 인덱스" (Ver 4.3 고유)
기존 형제(마젠타 토스톤)와 확실히 구별되는 독자 디자인.
- 흰/오프화이트 배경, 좌측 **큰 tabular 숫자**(02, 03…) + 우측 메뉴명 + chev, 행 사이 **헤어라인** 구분선.
- 전화번호부·디렉토리 무드의 에디토리얼 미니멀. 단일 액센트 컬러(잉크 네이비 계열), 여백 넉넉.
- 번호는 음성ARS 미러이므로 대/중/소 전 레벨 유지. 리프 행은 우측에 종착 태그(`음성 ARS`/`상담원`).
- 클래스 프리픽스 `av-`(ARS navigator), 목업 전역 CSS와 격리.

## 화면
1. **루트(대메뉴)**: 인사말 `안녕하세요`/`무엇을 도와드릴까요?`(2줄) + 설명 `원하시는 서비스를 선택하세요.` + 대메뉴 8개(넘버 인덱스 행).
2. **드릴다운(중/소)**: ‹뒤로 + 경로(브레드크럼) + 하위 메뉴 넘버 인덱스 리스트.
3. **음성ARS 안내 팝업**: 최종(리프, 비-직원연결) 도달 시. "'{리프명}' 서비스는 음성 ARS로 안내해 드려요" + [취소][연결하기].
4. **상담원 안내 팝업**: `0.직원연결` 리프. "'{상위 메뉴명}' 관련 상담원에게 연결할까요?" + [취소][연결하기].
5. **음성ARS 완료화면**: ((●)) 아이콘 + "음성 ARS로 연결하고 있어요" + "'{리프명}' 안내를 들려드려요" + [확인]→루트.
6. **상담원 완료화면**: 헤드셋 아이콘 + "상담원에게 연결하고 있어요" + "'{상위 메뉴명}' 관련 상담부서로 연결해 드려요" + [확인]→루트.

## 라우팅 규칙 (메뉴 행 클릭)
- `subs` 있음 → 드릴다운(경로 push).
- 리프(비-직원연결) → 음성ARS 안내 팝업(메뉴명 = 리프 자기 이름) → 연결 → 화면5.
- `0.직원연결` 리프 → 상담원 안내 팝업(상담분야 = **상위 메뉴명** = 브레드크럼 마지막) → 연결 → 화면6.
- 완료화면 [확인] → 루트 복귀(경로 초기화).
- ‹뒤로 → 경로 pop(루트에선 숨김).

## 상태 · 배선
- 상태: `av.path=[]`(IVR 인덱스 경로), `av.view`('list'|'done-voice'|'done-staff'), `av.leaf`(완료화면 표시명).
- 팝업은 `#dform2-phone` 내부 오버레이(av-pop-ov). 클릭 이벤트 위임(`#dform2-phone` 스코프).
- `window.__dform2Home` = 경로/뷰 리셋 후 루트 렌더(selectSian('dform2')에서 호출, 기존 배선 유지).
- dform2 인라인 스크립트를 dform 공용 forEach 스크립트에서 **분리**(독립 IIFE 신규). dform은 기존 스크립트 유지.
- app.js: dform2는 이미 isDarsView·SCHEME_META·selectSian 분기 등록됨 → 추가 배선 없음. SCHEME_META.dform2 라벨/kpi만 네비게이터 성격으로 갱신.

## 검증
헤드리스 CDP로 전 클릭 흐름 구동(루트→대/중/소 드릴다운→음성ARS 팝업/완료, 직원연결 팝업/완료, 뒤로/브레드크럼, 재진입 리셋) + 라이트 넘버 인덱스 비주얼 스크린샷 확인. 캐시버스트 갱신 후 커밋·푸시.
